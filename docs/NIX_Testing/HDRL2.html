def bpm_fit_compute(...)

bpm_fit_compute(image, exptime, [method, degree, pval, rel_chi_low, rel_chi_high, rel_coef_low, rel_coef_high])

A python wrapper around 'hdrl_bpm_fit_compute' function. 'hdrl_bpm_fit_parameter' is created inside the wrapper based on the keyword values.

Computes bad pixel mask from a set of flat images with a different exposure time. The algorithm performs fit on a stack of images and finds outliers using one of three defined methods.

Parameters

images : (N,N,N,) ndarray
Stack of images - along the third axis.
exptime : (N,) ndarray
Exptime corresponding to images. Should have same length as third axis of images.
method : str
Method used to determine bad pixels. If given method is none of the belowfunction returns Py_None. Accepted values are (default='pval'):
* 'pval'    : threshold on the p-values.
* 'rel_chi' : sigma cut on the relative chi values.
* 'rel_coef': sigma cut on the relative coefficients of polynomial fit.
degree : int, optional
Degree of polynomial to fit. (default=1)
pval : double, optional
Threshold in p-values. Enabled with pval (default=10.)
rel_chi_low : double, optional
Low threshold in relative chi values. Enabled with rel_chi (default=5.)
rel_chi_low : double, optional
Low threshold in relative chi values. Enabled with rel_chi (default=10.)
rel_coef_low : double, optional
Low threshold in relative coefficients. Enabled with rel_coef (default=5.)
rel_coef_low : double, optional
Low threshold in relative coefficients. Enabled with rel_coef (default=10.)

Returns

bp_mask : (N,N) ndarray
Output of hdrl_bpm_fit_compute converted to a numpy array. Good pixels are marked with 0 and bad pixels with 1. When rel_coef method used, algorithm computes bad pixels for each coefficient of the polynomial. Hence, the resulting bad pixel map is [2^0(0th)+2^1(1st)+2^2(2nd)+..] where bad pixel map derived from Nth order coefficients is given by (Nth). If the function fails, it returns Py_None instead of throwing an error.
